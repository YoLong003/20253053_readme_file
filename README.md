### Linux Process Management Commands

리눅스에서 효율적인 시스템 운영과 문제 해결을 위해서는 **프로세스를 확인하고 제어하는 능력**이 필수적입니다.  
그래서 저는 리눅스에서 가장 널리 사용되는 **프로세스 관리 명령어(Process Management Commands)**인:

- **`top`** — 실시간 프로세스 모니터링  
- **`ps`** — 정적 프로세스 조회  
- **`jobs`** — 백그라운드 셸 작업 확인  
- **`kill`** — 프로세스 종료 및 제어  

를 조사하고, 각 명령어의 기능과 사용법을 체계적으로 정리하였습니다.

## 1. top
### 개념
`top`은 시스템에서 실행 중인 프로세스와 자원(CPU, 메모리) 사용 상태를 **실시간으로 모니터링**하는 명령어입니다.

### 주요 기능
- CPU, 메모리 사용량 실시간 확인  
- 프로세스별 점유율 확인  
- PID 확인 가능  
- 실행 중인 프로세스 종료 가능(top 내에서 `k` 사용)

### 추가 옵션
- `top -u username` : 특정 사용자 프로세스만 보기  
- `top -b` : 배치 모드(로그 저장용)

### 예시
```bash
top
```

## 2. ps
`ps`는 현재 시스템에서 실행 중인 프로세스를 정적으로 조회하는 명령어입니다.
 **특정 시점의 프로세스 상태를 확인**할 때 사용됩니다.

### 주요기능
- 현재 실행 중인 프로세스 확인
- 프로세스 상태, PID, 메모리/CPU 사용량 확인
- 필터링 및 포맷 지정 가능
- 특정 프로세스를 찾아 PID 확인

### 추가 옵션
- `ps -e` : 모든 프로세스 표시
- `ps aux` : 가장 많이 쓰이는 전체 목록
- `ps -ef` : 상세한 전체 포맷 출력
- `ps -u username` : 특정 사용자의 프로세스만 보기

### 예시
```bash
ps aux
```

### 3. jobs
`jobs`는 현재 셸에서 실행 중인 백그라운드 작업을 확인하는 명령어입니다.
**현재 사용자가 실행한 셸 작업만 보여줍니다.**

### 주요 기능
- 백그라운드로 실행 중인 셸 작업 목록 확인  
- 작업 번호(Job ID)와 상태(Status) 확인  
- 작업 상태:  
  - `Running` : 실행 중  
  - `Stopped` : 중지됨  
  - `Done` : 완료됨

### 추가 옵션
- `jobs -l` : PID 포함하여 상세 정보 출력  
- `jobs -p` : PID만 출력  
- `jobs -r` : 실행 중(Running) 작업만 표시  
- `jobs -s` : 중지(Stopped) 작업만 표시

### 예시
```bash
sleep 100 &
jobs
 ```

## 4. kill
### 개념
`kill`은 프로세스의 **PID(Process ID)를 이용해 종료하거나 특정 시그널을 전송**하는 명령어입니다.  
주로 **프로세스 강제 종료**나 **정상 종료 요청**에 사용됩니다.

### 주요 기능
- PID를 이용한 프로세스 종료  
- 다양한 시그널 전송 가능  
- 시스템 자원 관리 및 문제 해결용으로 활용

### 추가 옵션(시그널)
- `kill -9 PID` : 강제 종료(SIGKILL)  
- `kill -15 PID` : 정상 종료(SIGTERM, 기본값)  
- `kill -l` : 사용할 수 있는 시그널 목록 확인  
- `kill -s SIGNAL PID` : 특정 시그널 전송

### 예시

 ```bash
 sleep 100 &
 jobs
 kill -9 %1
 jobs
 ```

## 5. 명령어 비교
| 명령어    | 실시간 모니터링 | 전체 프로세스 조회 | 백그라운드 작업 확인 | 종료 가능     | 특징                        |
| ------ | -------- | ---------- | ----------- | --------- | ------------------------- |
| `top`  | O        | O          | X           | O (top 내) | 실시간으로 CPU/메모리 사용량 확인 가능   |
| `ps`   | X        | O          | X           | X         | 특정 시점 프로세스 상태 조회 가능       |
| `jobs` | X        | X          | O           | X         | 현재 셸 작업 전용                |
| `kill` | X        | X          | X           | O         | PID 또는 Job ID로 프로세스 종료 가능 |



### 6. 결론

리눅스에서 프로세스를 효율적으로 관리하기 위해서는 ***`top`, `ps`, `jobs`, `kill`*** 명령어를 조합하여 사용하는 것이 가장 효과적입니다.  

- **`top`**: 실시간으로 CPU/메모리 사용량과 프로세스 상태 확인  
- **`ps`**: 특정 시점의 프로세스 상태 조회 및 PID 확인  
- **`jobs`**: 현재 셸에서 실행 중인 백그라운드 작업 확인  
- **`kill`**: PID나 Job ID를 이용한 프로세스 종료 및 제어  

이 네 가지 명령어를 이해하고 활용하면, 시스템 상태를 **모니터링 → 조회 → 제어**하는 전 과정을 능숙하게 수행할 수 있으며,  
문제가 있는 프로세스를 신속하게 종료하거나 백그라운드 작업을 효율적으로 관리할 수 있습니다. 


**핵심 요약**:  
- 모니터링: `top`  
- 조회: `ps`, `jobs`  
- 제어/종료: `kill`  
- 조합 사용으로 안정적 시스템 관리 가능
